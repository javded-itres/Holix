# Периодические задачи (Cron)

В Holix есть **встроенный планировщик** внутри **gateway**. Задачи хранятся в профиле и выполняются тем же агентом, что и чат — это не системный `crontab` и не отдельные скрипты.

Нужен запущенный gateway:

```bash
holix gateway start
holix gateway status
```

Файлы: `~/.holix/profiles/<profile>/data/cron/`. Лог запусков: `runs.log`.

См. также: [CLI.md](CLI.md#holix-cron), [SLASH_COMMANDS.md](SLASH_COMMANDS.md), навык `holix-cron`.

## Автосоздание из чата (0.1.16+)

Если в **Telegram**, **MAX** или **TUI** написать **повторяющийся** запрос обычным языком, Holix может создать cron-задачу **до** запуска агента — без `/cron add`.

**Примеры на русском:**

- `Присылай мне новости по теме X каждый день в 10 утра`
- `Каждые 2 часа проверяй статус gateway`
- `По будням в 9 утра присылай сводку по открытым задачам`

**На английском:**

- `Send me news about AI every day at 10 am`
- `Check disk space every 30 minutes`

Бот ответит id задачи, cron-выражение и `next_run_at` (UTC). Для Telegram/MAX задача привязывается к чату для уведомлений о результатах.

**Когда срабатывает автосоздание:**

- Явная периодичность (`каждый день`, `every day`, `каждые 30 минут`, `в 10 утра`, …)
- Осмысленный текст задачи после удаления фраз расписания (не короче ~8 символов)
- Не разовый запрос (`сейчас`, `один раз`, `just once`)
- Не вопрос про справку (`как настроить cron`, `/cron …`)

**Когда не срабатывает:**

- Сообщения с `/` в начале (используйте `/cron add`)
- Нет распознаваемого расписания
- Разовая задача без периодичности

Агент может вызвать инструмент **`schedule_cron`** (`schedule` + `task`), если авто-детект не сработал.

## Ручное создание

### Слэш (TUI / Telegram / MAX)

```
/cron add <расписание> :: <описание задачи>
```

Примеры:

```
/cron add every day at 9 :: Сводка по git за вчера
/cron add каждый день в 10 утра :: Новости по теме X
/cron add 0 9 * * 1-5 :: Подготовка к стендапу
/cron add hourly :: Проверка здоровья gateway
```

### CLI

```bash
holix cron add "every day at 9 :: Сводка по логам"
holix cron add "0 9 * * 1-5 :: Стендап" --name standup
holix cron list
holix cron enable <job-id>
holix cron disable <job-id>
holix cron remove <job-id>
```

## Форматы расписания

| Стиль | Примеры |
|-------|---------|
| Естественный язык (RU) | `каждый день в 10 утра`, `в 8 вечера`, `каждые 30 минут`, `каждый час`, `по будням` |
| Естественный язык (EN) | `every day at 9`, `every 30 minutes`, `hourly`, `daily`, `weekly`, `weekdays` |
| 5 полей cron | `0 9 * * *`, `*/15 * * * *`, `0 10 * * 1` |

Текст после `::` — **промпт агента** на каждый запуск. Укажите, что проверять, формат ответа, контекст workspace.

`next_run_at` в **UTC сервера** (croniter), если в промпте не оговорён часовой пояс.

## Управление задачами

| Действие | Слэш | CLI |
|----------|------|-----|
| Список / UI | `/cron` или `/cron list` | `holix cron list` |
| Включить | `/cron enable <id>` | `holix cron enable <id>` |
| Выключить | `/cron disable <id>` | `holix cron disable <id>` |
| Удалить | `/cron remove <id>` | `holix cron remove <id>` |
| Привязать сессию | `/cron bind <id>` | — |

- **TUI** — модальное окно со списком и кнопками.
- **Telegram** — inline-кнопки под `/cron`; в isolated mode не-админы видят только свои задачи.
- Id задач короткие; в CLI достаточно префикса.

---

## Как выполняется задача

- Планировщик тикает внутри **gateway** (~30 с); просроченные задачи стартуют в фоне.
- У каждого запуска свой `conversation_id`: `cron-<job-id>`.
- Агент получает контекст автоматического запуска и должен **завершить задачу и кратко отчитаться**.
- Поля: `last_run_at`, `last_status` (`success` / `error` / `running`), `next_run_at`, `run_count`.
- **Уведомления** в Telegram (`notify_chat_id`) или MAX (`notify_max_user_id` / `notify_max_chat_id`).

## Решение проблем

| Проблема | Что проверить |
|----------|----------------|
| Задача не запускается | `holix gateway status` — gateway должен работать |
| Автосоздание не сработало | Переформулируйте с явным расписанием; или `/cron add` / `schedule_cron` |
| Неверное расписание | `каждый день в 9` или корректный 5-field cron |
| Задача падает с ошибкой | `holix logs -s cron` или `runs.log` |
| Дубликаты | `/cron list`; удалите старую задачу |
| Шифрованный профиль | Для доступа к памяти нужен `HOLIX_UNLOCK_KEY` на gateway |
